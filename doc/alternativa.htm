    2. Arquitectura Restful

      3. La tecnología REST utiliza HTTP, lo que permite que pueda ser utilizada prácticamente por cualquier lenguaje de programación, además es un requisito de un servicio REST que el cliente y el servidor sean independientes entre sí.

      Entre sus caracteristicas más destacables tenemos que: las comunicaciones son más ligeras entre productor y consumidor, mantenibles y escalables; constituyéndose en un estilo de construcción popular para APIs basadas en la nube. A los servicios Web que utilizan la arquitectura REST, se les denominan API RESTful (Interfaces de programación de aplicaciones) o API REST.

      Operaciones http de API RESTful
      • GET para obtener un recurso
      • POST para crear un recurso en el servidor
      • PUT para cambiar el estado de un recurso o actualizarlo
      • DELETE para eliminar un recurso
