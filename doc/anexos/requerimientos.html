R2 Ingresar Parámetros

Categoria (Visible/No Visible)

Visible

TIPO (Necesario, Deseable)

Deseable

Prioridad

3

DESCRIPCION

Se adicionan parámetros, con la siguiente información:

  • código numérico
  • código alfanumérico
  • nombre del parámetro
  • grupo al cual pertenece el parámetro, es decir, el grupo que puede ser medido con el parámetro que se está agregando.
  • unidad
  • código utilizado en el IDEAM para dicho parámetro.
  • Un carácter indicando si se calcula valores multianuales para ese parámetro
  • Número de decimales utilizados para este parámetro en los reportes.
  • Un carácter Y o N indicando si el número de decimales es fijo o variable.
  • Ancho: número total de posiciones que ocupa cualquier dato, independientemente del número de decimales que tenga. Al ancho debe sumarse una posición para el flag del dato.
  • Descriptor.
  • Tabla. Indica el nombre de la tabla en la cual se encuentran los datos. Un contenido nulo del campo indica que los datos se encuentran en la tabla estándar de acuerdo a la frecuencia del parámetro (horario, diario o mensual). Esta opción tiene especial importancia para garantizar la expansión de la base de datos, fraccionando las tablas con grandes volúmenes.

Criterios de Aceptación

Ø       El sistema debe permitir crear un nuevo parámetro

Ø       El sistema debe permitir la consulta de la información del parámetro para verificar que ha quedado correctamente registrado en la base de datos.

Ø       El sistema debe permitir modificar la información del parámetro.

Ø       El sistema debe permitir eliminar un parámetro solamente en el caso que no exista ningún dato relacionado con este.

R6.1 Ingresar observaciones

Categoria (Visible/No Visible)

Visible

TIPO (Necesario, Deseable)

Necesario

Prioridad

5

DESCRIPCION

Se debe poder digitar el dato o los datos de las observaciones realizadas en un elemento de un grupo para un determinado parámetro.  Los datos observados se agregan con el flag O (Observado). Después de haber adicionado un dato el cual resulta ser inconsistente este se debe modificar con el flag d (Dudoso).

Existe una opción para verificar los datos que se están introduciendo, comparándolos con los datos multianuales existentes de cada parámetro, por ejemplo, si se están digitando los datos de temperatura de un lugar a las 7 de la mañana, el sistema puede proporcionar los valores extremos que ha presentado esa estación para muchos años de registro, si el valor digitado esta fuera de este rango el sistema debe mostrar una alerta.

Criterios de Aceptación

Ø       El sistema debe permitir registrar una o varias observaciones

Ø       El sistema debe permitir consultar las observaciones registradas para verificar que fueron almacenadas correctamente en la base de datos.

Ø       El sistema debe permitir modificar la información de las observaciones. 

Ø       El sistema no debe permitir eliminar las observaciones (a un usuario no autorizado)

Ø       Si la opción Verificar con Multianuales está activa, el sistema debe confrontar el dato digitado con el valor multianual del parámetro para el mes respectivo. Si el dato ingresado esta fuera del rango de valores extremos de la serie, el sistema debe producir una alerta.

R6.3 Ingresar datos de formatos

Categoria (Visible/No Visible)

No Visible

TIPO (Necesario, Deseable)

Necesario

5

DESCRIPCION

Mediante procedimientos de transferencia de datos, se deben pasar los datos almacenados en archivos magnéticos bajo la estructura EAAB o IDEAM, a la base de datos del sistema, interpretando los respectivos formatos de ingreso.

El formato de la EAAB es el siguiente (datos horarios):

 

o211951000120000601 1.00   4.031

o211951000120000601 2.00   0.737

o211951000120000601 3.00   0.732

o211951000120000601 4.00   0.735

o211951000120000601 5.00   0.745

o211951000120000601 6.00   0.758

o211951000120000601 7.00   0.774

o211951000120000601 8.00   0.776

o211951000120000601 9.00   0.768

o21195100012000060110.00   0.761

 

o =  (letra o) indica número de observaciones en un día al cual pertenece el dato o fila.

2119510 = código estación

001 = Código del parámetro

20000601 = fecha

1.0     = Hora decimal

4.031 = dato

(blanco) = flag

 

Para los datos diarios se suprime la hora y el primer byte sirve para indicar si proviene de una serie horaria. Para los datos mensuales además de suprimirse la hora se suprime el día.

Existe una opción para verificar los datos que se están introduciendo, comparándolos con los datos multianuales existentes de cada parámetro, por ejemplo, si se están digitando los datos de temperatura de un lugar a las 7 de la mañana, el sistema puede proporcionar los valores extremos que ha presentado esa estación para muchos años de registro, si el valor digitado esta fuera de este rango el sistema debe mostrar una alerta.

Criterios de Aceptación

Ø       El sistema debe permitir el ingreso de los datos, previa la verificación del respectivo formato presente en la estructura de los datos a ser transferidos.

Ø       Si uno o más datos ya existen, el sistema debe pedir aceptación para reemplazar uno a uno o todos los datos.

Ø       El sistema debe permitir consultar los datos ingresados.

Ø       Si la opción Verificar con Multianuales está activa, el sistema debe confrontar el dato digitado con el valor multianual del parámetro para el mes respectivo. Si el dato ingresado esta fuera del rango de valores extremos de la serie, el sistema debe producir una alerta.

R7.7 Obtener automáticamente los caudales respectivos a un periodo seleccionado

Categoria (Visible/No Visible)

No Visible

TIPO (Necesario, Deseable)

Necesaria

Prioridad

5

DESCRIPCION

El sistema permite calcular los caudales de una corriente, a partir de los datos de nivel almacenados en el módulo Horarios. Dada una estación y un periodo dado, el sistema utiliza cada dato de nivel para estimar el caudal correspondiente. Para ello selecciona la tabla o ecuación en el sistema para esa estación, que este vigente para la fecha del dato de nivel. Ø       En el caso de tabla, el dato de nivel no debe ser inferior al mínimo nivel registrado para la tabla ni superior al máximo nivel de la tabla.

Ø       En el caso de una ecuación, el dato de nivel debe estar dentro del rango de validez de uno de los tramos (ecuaciones) de la curva.

Ø       Debe verificarse que el usuario seleccione solamente uno de los dos métodos para el cálculo de los caudales: tablas o ecuaciones.
